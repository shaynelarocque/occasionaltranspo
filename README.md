# Ottawa–Gatineau Transit Travel Time Map

An interactive map that distorts geography so stop distances reflect actual travel time, not physical distance. Inspired by [Nate Parrott's NYC subway map](https://github.com/nate-parrott/subway).

Click any stop to see all other stops repositioned based on how long it takes to reach them using OC Transpo + STO buses and the O-Train.

---

## Setup

### 1. Get the GTFS data

**STO (Gatineau) — auto-downloaded**
The script downloads this automatically. No action needed.

**OC Transpo (Ottawa) — manual download required**

1. Register for a free API key at https://nextrip-public-api.developer.azure-api.net/
2. After registration, download the **GTFS Schedule** zip file from the portal
3. Save it as `data/octranspo_gtfs.zip`

> You can also try finding a recent OC Transpo GTFS on the
> [Mobility Database](https://mobilitydatabase.org/feeds/mdb-738) or
> [Transitland](https://www.transit.land/feeds/f-f24-octranspo).

### 2. Process the GTFS data

```bash
python3 process_gtfs.py
```

This generates:
- `subway.js` — stop locations and route shapes
- `schedules/transfers.js` — transfer connections between stops
- `schedules/weekday_rush.json` — events 7–10 AM weekday
- `schedules/weekday_evening.json` — events 7–10 PM weekday
- `schedules/weekend_afternoon.json` — events 12–3 PM weekend

### 3. Serve and open

```bash
python3 -m http.server 8000
```

Open http://localhost:8000 in your browser.

> The app must be served over HTTP (not opened as a file) because it loads JSON via `fetch`.

## How it works

The algorithm (`virtual_rider.js`) is event-based:

1. You click a stop. The algorithm starts you there at the schedule's start time.
2. It iterates through every departure event in the time window.
3. For each event, it models: if you're already at that stop (or can walk there via a transfer), you can board the vehicle. If you're on a vehicle that's arriving at a stop, you can exit.
4. Each stop accumulates the earliest possible arrival time from your origin.
5. The map then repositions every stop so its distance from the center equals its travel time.

Bus routes with infrequent service will naturally show as "far away" because you may wait a long time for the next departure.

## Deploying to GitHub Pages

The generated files (`subway.js`, `schedules/*.json`, `schedules/transfers.js`) are in `.gitignore` by default because they're large and regenerated from source data. To publish on GitHub Pages you need them in the repo.

### Option A — commit the generated files (simplest)

1. Edit `.gitignore` and remove (or comment out) these three lines:
   ```
   data/
   schedules/*.json
   subway.js
   ```
2. Commit the generated files:
   ```bash
   git add subway.js schedules/
   git commit -m "Add generated data files for GitHub Pages"
   git push
   ```
3. In your repo on GitHub: **Settings → Pages → Source → Deploy from branch → `main` / `(root)`**.

Your map will be live at `https://<your-username>.github.io/<repo-name>/`.

> **Note:** The `data/` folder (raw GTFS zips) is still excluded — those are large and not needed to serve the app. Only add the generated output files.

### Option B — GitHub Actions (automatic regeneration)

If you want the map to rebuild automatically whenever you push, create `.github/workflows/build.yml`:

```yaml
name: Build & Deploy

on:
  push:
    branches: [main]
  schedule:
    - cron: '0 6 * * 1'   # rebuild every Monday at 6 AM UTC

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Download OC Transpo GTFS
        env:
          OCTRANSPO_GTFS_URL: ${{ secrets.OCTRANSPO_GTFS_URL }}
        run: |
          mkdir -p data
          curl -L "$OCTRANSPO_GTFS_URL" -o data/octranspo_gtfs.zip

      - name: Process GTFS
        run: python3 process_gtfs.py

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: .
          exclude_assets: 'data/,.git/,.github/'
```

Add the OC Transpo GTFS download URL as a repository secret named `OCTRANSPO_GTFS_URL` (Settings → Secrets → Actions).

## Performance notes

With all routes included, the schedule JSON files can be 5–20 MB. If loading is slow:

- Reduce the time window in `SCHEDULE_CONFIGS` in `process_gtfs.py` (e.g., 2 hours instead of 3)
- Or filter routes by type: only include O-Train (type 1) and rapid bus routes
